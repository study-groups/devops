TSM - Tetra Service Manager

TSM manages long-running processes defined in .tsm files. Each service runs in its own process with automatic restart on failure.

tsm start devwatch # Start a service tsm stop devwatch # Stop a service tsm restart devwatch # Restart a service tsm ls # List all services

Service Files

Services are defined in .tsm files at $TETRA_DIR/services/:

# devwatch.tsm port=3010 script=server/index.js cwd=$TETRA_SRC/devwatch

Logging

Each service logs to $TETRA_DIR/run/<service>/current.log

  • [L] button - Watch service logs in Logs panel
  • Expand row - See recent logs inline
  • logs button - Jump to full Logs panel
tsm logs devwatch # Tail logs tsm logs devwatch -n 100 # Last 100 lines tsm logs devwatch -f # Follow mode

Remote Execution

TSM commands run on remote servers via SSH when environment is not local.

Configure in tetra.toml:

[env.dev] host = "dev.example.com" user = "deploy" [env.prod] host = "prod.example.com" user = "root"

Environment Model

org (tetra, nh, pj) └── env (local, dev, staging, prod) └── services (devwatch, pbase, ...)

Select org in header, env per-panel. Remote envs use SSH to execute TSM commands on configured hosts.

Log Aggregation

The [L] button broadcasts a log-watch-change message via Terrain.Bus. The Logs panel subscribes and filters its view to show only watched services.

TSM Panel Terrain.Bus Logs Panel │ │ │ │ click [L] │ │ ├───────────────────►│ │ │ log-watch-change │ │ │ ├───────────────────►│ │ │ │ │ │ filter by service

Patrol Mode

TSM can automatically restart crashed services:

tsm patrol # Run continuously tsm patrol --once # Single check pass

Health & Diagnostics

tsm doctor # System health check tsm info svc # Detailed service info

Use Terrain.State.apiUrl('/api/foo') to auto-append context params.

Tetra Console Overview

The Tetra Console is a multi-panel dashboard for managing tetra instances. It uses iframes to isolate each panel, with a shared message bus for communication.

Key Concepts

Terrain.Bus
Pub/sub message bus connecting all panels. Use Terrain.Bus.subscribe('*', fn) to listen, Terrain.Bus.publish({type, ...}) to send.
Terrain.State
Shared state for org/env/user context. Auto-syncs across panels via env-change messages.
Terrain.Iframe
Iframe helpers: init() to setup, send() to message parent, on(action, fn) for DOM delegation.

Admin Panel Purpose

This admin panel is for administering both local and remote tetra instances:

  • Local: Manage captures, view timings, check API health on your dev machine
  • Remote: When env is set to prod/staging, commands are executed via SSH to the remote host defined in tetra.toml

Context (org/env/user)

  • org: Organization/project namespace (e.g., tetra, myproject)
  • env: Environment target (local, prod, staging)
  • user: SSH user override for remote connections

The context selector in the header broadcasts changes to all panels.

API Caching

TSM API uses a 5-second TTL cache for expensive operations like /api/tsm/ls. The "Cached" indicator shows if last response was from cache. Use "Refresh All" to force fresh data.

Capture Grooming

Captures are screenshots/DOM snapshots stored in $TETRA_DIR/orgs/{org}/captures/. Use the grooming controls to clean up old captures by age.
